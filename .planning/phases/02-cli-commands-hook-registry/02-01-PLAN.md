<task type="auto" tdd="true">
  <name>Task 1: Implement add and remove commands with tests</name>
  <files>src/commands/add.ts, src/commands/remove.ts, tests/commands/add.test.ts, tests/commands/remove.test.ts</files>
  <behavior>
    - add: given a valid hook name from registry, copies script to hooksDir, chmod +x, calls applyMerge to add settings entry
    - add: given an unknown hook name, prints error and exits
    - add: with --dry-run, prints what would happen without writing
    - add: skips if hook already installed (applyMerge dedup)
    - remove: given an installed hook name, deletes script file from hooksDir, removes matching entry from settings.json hooks
    - remove: given a hook not installed, warns and exits cleanly
    - remove: supports --dry-run
  </behavior>
  <action>
    Create `src/commands/add.ts`:
    - Export `_addAt(opts: { settingsPath, hooksDir, hookName, dryRun? })` for testability (same pattern as _initAt)
    - Look up hook in registry via `getHook(hookName)`. If not found, `log.error` and return
    - Resolve source script path: `path.join(__dirname, '..', '..', 'registry', 'hooks', hook.scriptFile)` using `import.meta.url` + `fileURLToPath`
    - Copy script to `path.join(hooksDir, hook.scriptFile)` using `fs.copyFile`
    - Run `fs.chmod(destPath, 0o755)` to make executable
    - Build `newHooks` array from hook metadata: `[{ event: hook.event, matcher: hook.matcher, hook: { type: 'command', command: destPath } }]`
    - Call `applyMerge({ settingsPath, newHooks, dryRun })`
    - Print summary: what was installed, which pack it belongs to
    - Export `addCommand(opts: { scope, hookName, dryRun? })` as the public entry that resolves paths from scope

    Create `src/commands/remove.ts`:
    - Export `_removeAt(opts: { settingsPath, hooksDir, hookName, dryRun? })` for testability
    - Look up hook in registry via `getHook(hookName)` to get scriptFile and event/matcher
    - Check if script exists at `path.join(hooksDir, hook.scriptFile)`. If not, warn "not installed"
    - Read settings, find and remove the matching hook group entry (match by command path containing hook.scriptFile)
    - If not dry-run: delete the script file (`fs.unlink`), backup then write updated settings
    - Print summary of what was removed
    - Export `removeCommand` as public entry

    Tests use `tmp` directories (vitest `beforeEach` with `mkdtemp`). Create a fake registry hook script file in tmp to simulate the bundled script source. Mock the registry module's `getHook` if needed, or create a real registry.json in tmp.
  </action>

    <automated>cd /Users/admin/workspace/claude-hooks && npx vitest run tests/commands/add.test.ts tests/commands/remove.test.ts --reporter=verbose</automated>
  </verify>
  <done>
    - `_addAt` copies script, sets +x, merges settings entry. Test proves script exists and is executable, settings.json has the hook entry.
    - `_addAt` with unknown hook name does not throw, prints error.
    - `_addAt` with dry-run does not create files.
    - `_removeAt` deletes script and removes settings entry. Test proves both are gone.
    - `_removeAt` with non-installed hook warns without error.
  </done>
