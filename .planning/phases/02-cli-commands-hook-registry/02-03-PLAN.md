<task type="auto" tdd="true">
  <name>Task 1: Implement doctor command</name>
  <files>src/commands/doctor.ts, tests/commands/doctor.test.ts</files>
  <behavior>
    - doctor with valid setup (scripts exist, executable, settings valid): all checks pass with green output
    - doctor with missing script file: reports which scripts are missing
    - doctor with non-executable script: reports which scripts lack +x
    - doctor with invalid settings.json (malformed JSON): reports parse error
    - doctor with orphaned settings entry (references script that doesn't exist): reports orphan
    - doctor with conflicting hooks (same event+matcher, different commands): warns about conflicts (SET-05)
    - doctor with no hooks installed: reports clean state
  </behavior>
  <action>
    Create `src/commands/doctor.ts`:
    - Export `_doctorAt(opts: { settingsPath: string; hooksDir: string })` for testability
    - Run these checks in order, collecting results:

    **Check 1: Settings file exists and is valid JSON**
    - Try readSettings(settingsPath)
    - If file missing: warn (not error — user may not have initialized)
    - If malformed JSON: error

    **Check 2: Hooks directory exists**
    - Check existsSync(hooksDir)

    **Check 3: Installed scripts exist and are executable**
    - For each hook entry in settings.hooks, extract the command path
    - Check file exists with `existsSync`
    - Check executable with `accessSync(path, constants.X_OK)`
    - Report missing or non-executable scripts

    **Check 4: No orphaned settings entries**
    - For each hook entry in settings, check if the referenced script file exists
    - Orphan = settings entry pointing to non-existent script

    **Check 5: No conflicting hooks (SET-05)**
    - Collect all (event, matcher) tuples from settings.hooks
    - If same (event, matcher) appears with different commands, warn about conflict
    - This is a warning, not an error

    Output format:
    ```
    claude-hooks doctor

    [PASS] Settings file valid
    [PASS] Hooks directory exists
    [FAIL] Script not found: .claude/hooks/missing-hook.sh
    [PASS] No orphaned entries
    [WARN] Conflicting hooks: PreToolUse/Edit|Write has 2 commands

    3 passed, 1 failed, 1 warning
    ```

    Use picocolors: green for PASS, red for FAIL, yellow for WARN.

    Export `doctorCommand(opts: { scope })` as public entry.

    Tests: create various broken states in tmp dirs (missing scripts, bad JSON, orphans) and verify doctor reports them correctly.
  </action>
  <verify>
    <automated>cd /Users/admin/workspace/claude-hooks && npx vitest run tests/commands/doctor.test.ts --reporter=verbose</automated>
  </verify>
  <done>
    - doctor detects missing scripts, non-executable scripts, invalid JSON, orphans, and conflicts
    - doctor with clean setup shows all PASS
    - Output uses colored PASS/FAIL/WARN labels
    - Returns exit code 1 if any FAIL, 0 if only PASS/WARN
  </done>
