Hook input (stdin):
```json
{
  "session_id": "abc123",
  "transcript_path": "/path/to/transcript.jsonl",
  "cwd": "/project/dir",
  "hook_event_name": "PreToolUse",
  "tool_name": "Bash",
  "tool_input": {
    "command": "npm test",
    "description": "Run tests",
    "file_path": "/path/to/file.ts",
    "content": "file content..."
  }
}
```

Hook output (stdout, optional):
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",
    "permissionDecisionReason": "Blocked: reason here"
  }
}
```

Exit codes: 0 = allow, 2 = block (stderr shown to Claude), other = non-blocking error

<task type="auto">
  <name>Task 1: Create security and quality pack scripts (4 hooks)</name>
  <files>registry/hooks/sensitive-path-guard.sh, registry/hooks/exit-code-enforcer.sh, registry/hooks/post-edit-lint.sh, registry/hooks/ts-check.sh</files>
  <action>
    Create `registry/hooks/` directory.

    **sensitive-path-guard.sh** (PreToolUse, matcher: Edit|Write):
    - Read JSON from stdin into variable
    - Extract `tool_input.file_path` (or `tool_input.content` for Write which includes path)
    - Check against blocked patterns: `.env`, `.env.*`, `*credentials*`, `*secret*`, `*.key`, `*.pem`, `id_rsa`, `id_ed25519`, `.claude/settings.json` (prevent hooks from editing their own config)
    - Use bash pattern matching (case statement or grep), NOT regex (POSIX compat)
    - If blocked: write reason to stderr, exit 2
    - If allowed: exit 0
    - Include inline comments explaining each blocked pattern

    **exit-code-enforcer.sh** (PreToolUse, matcher: Bash):
    - Read JSON from stdin
    - Extract `tool_input.command`
    - Check for dangerous command patterns: `rm -rf /`, `rm -rf ~`, `rm -rf $HOME`, `chmod 777`, `> /dev/sda`, `:(){:|:&};:` (fork bomb), `dd if=`
    - If dangerous: stderr message, exit 2
    - If safe: exit 0

    **post-edit-lint.sh** (PostToolUse, matcher: Write|Edit):
    - Read JSON from stdin
    - Extract `tool_input.file_path`
    - Detect file type by extension: .ts/.tsx -> eslint or biome, .py -> ruff, .sh -> shellcheck
    - Run the appropriate linter if available (check with `command -v`)
    - If linter not found, exit 0 silently (don't block for missing tools)
    - Output lint results to stderr for Claude to see (informational, not blocking)
    - Always exit 0 (PostToolUse hooks inform, they don't block)

    **ts-check.sh** (PostToolUse, matcher: Write|Edit):
    - Read JSON from stdin
    - Extract `tool_input.file_path`
    - Only run if file is .ts or .tsx
    - Check if `npx tsc` is available, if not exit 0
    - Run `npx tsc --noEmit --pretty 2>&1` and capture output
    - Output type errors to stderr for Claude to see
    - Always exit 0 (informational)

    All scripts: `#!/bin/bash` shebang, no bashisms beyond what bash 3.2 supports (no `declare -A`, no `|&`, no `${var,,}`). Use `printf` over `echo -e`. Parse JSON with grep/sed (no jq dependency -- keep zero external deps).
  </action>

<task type="auto">
  <name>Task 2: Create cost and error pack scripts (3 hooks) and test all 7</name>
  <files>registry/hooks/web-budget-gate.sh, registry/hooks/cost-tracker.sh, registry/hooks/error-advisor.sh, tests/hooks/scripts.test.ts</files>
  <action>
    **web-budget-gate.sh** (PreToolUse, matcher: WebSearch|WebFetch):
    - Read JSON from stdin
    - Extract `session_id` for tracking
    - Use a counter file at `/tmp/claude-hooks-web-count-$session_id`
    - Read current count (default 0), increment
    - If count > configurable limit (default 10, read from env `CLAUDE_HOOKS_WEB_LIMIT`), stderr "Web budget exceeded", exit 2
    - Otherwise write new count, exit 0

    **cost-tracker.sh** (PostToolUse, no matcher):
    - Read JSON from stdin
    - Extract `tool_name`, `session_id`, timestamp
    - Append one line to `/tmp/claude-hooks-cost-$session_id.log`: `timestamp|tool_name`
    - Always exit 0 (purely informational logging)

    **error-advisor.sh** (PostToolUse, matcher: Bash):
    - Read JSON from stdin
    - This is a PostToolUse hook, so check if the tool execution failed
    - Extract `tool_input.command` and look for common error patterns in the tool output
    - Check for known error patterns and suggest fixes via stderr:
      - "EADDRINUSE" -> "Port already in use. Try: lsof -i :<port> | kill"
      - "ENOENT" -> "File not found. Check the path exists."
      - "permission denied" -> "Try: chmod +x <file> or check ownership"
      - "MODULE_NOT_FOUND" -> "Try: npm install"
      - "ENOMEM" -> "Out of memory. Close other processes."
    - Always exit 0 (advisory only)

    **tests/hooks/scripts.test.ts**:
    - Integration tests that feed mock JSON via stdin to each script and verify exit code
    - Use Node `child_process.execSync` or `spawn` to run each script
    - Test cases per script:

      - exit-code-enforcer: "rm -rf /" -> exit 2, "npm test" -> exit 0
      - web-budget-gate: first call -> exit 0, 11th call -> exit 2 (use unique session_id per test)
      - cost-tracker: any input -> exit 0, log file created
      - error-advisor: any input -> exit 0
      - post-edit-lint: any input -> exit 0
      - ts-check: any input -> exit 0
    - Clean up /tmp files in afterEach

    All scripts must be `chmod +x` after creation.
